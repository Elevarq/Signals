# Arq Signals v0.2.0 — First Public Release

Arq Signals is a PostgreSQL telemetry collector designed to safely extract operational signals from running PostgreSQL systems.

This release marks the first public open-source release of the Arq Signals collector.

The project focuses on **safe diagnostics, early failure detection, and production-grade observability signals**.

## Highlights

- 29 production-safe collectors
- PostgreSQL 14–18 compatibility
- Read-only architecture
- Graceful degradation when permissions are limited
- CLI and API interfaces
- STDD-based development methodology
- 135 automated tests

## Operational Signals Detected

Arq Signals collects signals related to critical database stability risks:

- transaction ID wraparound risk
- autovacuum failures
- stale replication slots
- WAL pressure
- connection saturation
- long-running transactions
- checkpoint pressure
- database growth signals

These signals help operators detect database instability **before outages occur**.

## Safety Model

Arq Signals is designed to operate safely in production environments.

Key safety guarantees:

- Runs with read-only transactions
- Rejects superuser roles by default
- Detects unsafe role overrides
- Never writes to the monitored database
- Gracefully degrades when privileges are limited

Unsafe overrides are explicitly recorded in export metadata.

## STDD Methodology

Arq Signals is developed using **Specification & Test-Driven Development (STDD)**.

This means:

- behavior is defined in specifications
- acceptance tests define correctness
- implementations must satisfy those tests
