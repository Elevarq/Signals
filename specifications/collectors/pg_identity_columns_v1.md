# pg_identity_columns_v1 ΓÇö Collector Specification

## Purpose

Per-column identity / `SERIAL` metadata for user-schema tables.
Surfaces the catalog signals for the identity-column migration rule
`data_model.identity_column_migration.v1`
so downstream analysis can distinguish `SERIAL` / `BIGSERIAL` columns from
`GENERATED ALWAYS AS IDENTITY` and `GENERATED BY DEFAULT AS
IDENTITY` columns, and so it can locate the owning sequence for
the migration DDL.

Without this collector the rule degrades silently to no-finding;
the existing `pg_columns_v1` collector emits only a generic
`has_default` boolean and does not surface `attidentity` or the
`pg_depend` link from a column to its owning sequence.

## Catalog source

- `pg_attribute` (`attidentity`, `atthasdef`, `atttypid`, `attnum`)
- `pg_class` (relation identity and kind)
- `pg_namespace` (schema name)
- `pg_type` (base type name; `typname`)
- `pg_depend` (auto-ownership dependency linking a column to its
  owning sequence; `deptype = 'a'`)
- `pg_constraint` (primary-key + unique-constraint membership)

## Output columns

| Column | Type | Description |
|---|---|---|
| schemaname | text | Schema of the relation (`pg_namespace.nspname`) |
| relname | text | Table name (`pg_class.relname`) |
| attname | text | Column name |
| atttypname | text | Base type name (`pg_type.typname`) ΓÇö e.g. `int2`, `int4`, `int8`, `uuid` |
| attidentity | text | `''` (not identity), `'a'` (GENERATED ALWAYS), `'d'` (GENERATED BY DEFAULT) |
| atthasdef | bool | True when the column has a `DEFAULT` expression |
| default_is_nextval | bool | True when the column has a default AND there is an auto-owned sequence dependency (the SERIAL fingerprint) |
| sequence_schema | text | Schema of the owning sequence (when `auto_owned_sequence`) |
| sequence_name | text | Name of the owning sequence (when `auto_owned_sequence`) |
| auto_owned_sequence | bool | True when `pg_depend.deptype = 'a'` links the column to a sequence |
| is_primary_key | bool | Column is part of the table's primary-key constraint |
| is_unique | bool | Column is part of a unique constraint |

`default_is_nextval` is derived from the auto-owned-sequence
relationship rather than from the parsed default expression. This
keeps the collector free of `pg_get_expr` (which would risk
exposing literal default values from non-`SERIAL` columns and is
the reason `pg_columns_v1` does not emit the expression text
either). For every real `SERIAL` / `BIGSERIAL` / `SMALLSERIAL`
column the auto-ownership link is present by construction; a
manually-defined sequence + `DEFAULT nextval(...)` binding
without `OWNED BY` is the only false-negative path and is rare
in practice.

## Row-emission filter

A row is emitted when the column satisfies any of:

- `attidentity <> ''` (column is a `GENERATED ... AS IDENTITY`
  column), OR
- the column has an auto-owned sequence dependency (the SERIAL
  fingerprint), OR
- the column's base type is one of `int2`, `int4`, `int8`,
  `uuid` (potential surrogate primary key ΓÇö even when the
  column has no default today, the analyzer rule may need it as
  context for adjacent recommendations).

Columns that don't match any of the above are not emitted. This
keeps the row count modest on schemas with many wide text /
jsonb columns while preserving every row the
`data_model.identity_column_migration.v1` rule may need.

## Relation kinds included

- `r` ΓÇö regular table
- `p` ΓÇö partitioned table

Views, materialized views, and foreign tables are excluded ΓÇö
they cannot carry `IDENTITY` columns.

## Schema filter

Standard exclusion: `pg_catalog`, `information_schema`,
`pg_toast`, `pg_temp_%`, `pg_toast_temp_%`. Uses the shared
`SchemaFilter` constant.

## Invariants

- Deterministic ordering:
  `ORDER BY schemaname, relname, attnum`.
- Empty result serializes as `[]`.
- Stable output column order (explicit `SELECT`, no `SELECT *`).
- Read-only query, passes linter.
- No default expression text in output. `default_is_nextval` is
  a boolean derived from the auto-ownership link.
- One row per (schema, table, column) tuple. If a column is
  part of both the primary key AND a separate unique
  constraint, the row is emitted once with both
  `is_primary_key=true` and `is_unique=true`.

## Configuration

- Category: `schema`
- Cadence: 24h (`CadenceDaily`)
- Retention: `RetentionMedium`
- Min PG version: 10 (`attidentity` and `GENERATED AS IDENTITY`
  landed in PostgreSQL 10; on earlier versions the column will
  always be empty and the collector will still work, but the
  analyzer rule's PG-10 gate makes a lower `MinPGVersion` value
  irrelevant)
- Enabled by default: yes
- Sensitivity: low ΓÇö column names, type names, and
  primary/unique-constraint membership are structural metadata
  also exposed by existing collectors (`pg_columns_v1`,
  `pg_constraints_v1`). No literal values or default
  expression text are emitted.

## Analyzer use cases

- `data_model.identity_column_migration.v1` ΓÇö distinguishes
  `SERIAL` from `IDENTITY` columns, identifies the owning
  sequence for the migration DDL, and confirms primary-key
  membership for the Pattern A / Pattern B branching.
- Future schema/data-model rules that need to reason about
  identity-column distribution across the catalog.

## Out of scope

- INSERT-pattern detection for the Pattern B (`BY DEFAULT` ΓåÆ
  `ALWAYS`) recommendation. That guard reads
  `pg_stat_statements` and is an analyzer-side concern.
- Sequence current value, increment, cycle behaviour. Surfaced
  by the existing `pg_sequences_v1` collector; the analyzer
  joins the two collectors on `(sequence_schema, sequence_name)`
  to obtain `last_value` for the migration DDL's `START WITH`.
- Default expression text. The `default_is_nextval` boolean is
  the only default-related signal emitted.

## Test plan

- Registration + linter clean.
- Cadence is `CadenceDaily`.
- SQL excludes system schemas.
- SQL contains `ORDER BY`.
- SQL does not contain `SELECT *`.
- SQL does not contain `pg_get_expr` or any reference to the
  default expression text (the `default_is_nextval` boolean
  must be derived from the `pg_depend` link, not from parsing
  the default).
- Output columns match the spec table above (12 columns).
- SQL joins `pg_depend` with `deptype = 'a'` for the
  auto-ownership link.
- SQL joins `pg_constraint` for `is_primary_key` /
  `is_unique` membership using `unnest(conkey)`.
- Relation-kind filter restricts to `r` and `p`.
